🚀 Déploiement Production : a11y-formation

Ce guide détaille la mise en service sécurisée de l'application sur un VPS Rocky Linux 9 (IONOS).

Note de sécurité : Ne lancez jamais ces commandes en mode root pur sans comprendre l'impact. Utilisez un utilisateur sudo dédié.
ÉTAPE 0

Initialisation de l'Accès & Sudoers

Sur un VPS IONOS, vous vous connectez souvent en root la première fois. Il est impératif de créer un utilisateur dédié pour ne pas compromettre tout le système.

# 1. Créer l'utilisateur (remplacez 'deployuser' par votre nom)
adduser deployuser

# 2. Définir un mot de passe robuste
passwd deployuser

# 3. Ajouter aux sudoers (autorise les commandes d'administration)
usermod -aG wheel deployuser

# 4. Tester la connexion (Ouvrez un NOUVEAU terminal sur votre PC)
ssh deployuser@VOTRE_IP_VPS
# Puis testez le sudo :
sudo whoami # Doit répondre 'root'
L'utilisateur fait maintenant partie du groupe wheel, l'équivalent de sudo sur Rocky/RHEL.
ÉTAPE 1

Préparation du Système (OS hardening)

Connectez-vous à votre VPS et commencez par sécuriser les bases de Rocky 9.

# Mise à jour complète
sudo dnf update -y

# Installation des outils de base
sudo dnf install -y yum-utils device-mapper-persistent-data lvm2 git curl

# Désactiver SELinux (optionnel mais recommandé pour Docker si vous débutez)
sudo setenforce 0
sudo sed -i 's/SELINUX=enforcing/SELINUX=permissive/g' /etc/selinux/config
ÉTAPE 2

Installation de Docker & Compose

Rocky 9 vient avec Podman par défaut. Pour notre stack, nous installons Docker CE officiel.

# Supprimer les conflits
sudo dnf remove -y podman buildah

# Ajouter le repo Docker
sudo dnf config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# Installer Docker
sudo dnf install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Lancer et activer au boot
sudo systemctl enable --now docker

# Ajouter votre utilisateur au groupe docker (éviter le sudo systématique)
sudo usermod -aG docker $USER
# Appliquer le changement de groupe :
newgrp docker
ÉTAPE 3

Configuration Firewall (Firewalld)

C'est ici que la sécurité réseau se joue. On ne laisse ouvert que le strict nécessaire. Sur certaines installations minimales de Rocky 9, firewalld n'est pas installé par défaut.

# 1. Installer le service firewall
sudo dnf install -y firewalld

# 2. Lancer et activer le service
sudo systemctl enable --now firewalld

# 3. Autoriser HTTP, HTTPS et SSH
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https
sudo firewall-cmd --permanent --add-service=ssh

# 4. Appliquer
sudo firewall-cmd --reload

# Vérifier
sudo firewall-cmd --list-all
ÉTAPE 4

Déploiement de l'Application

Nous utilisons Git pour récupérer le code et Docker Compose pour orchestrer les services.

# Cloner le repo
git clone [URL_DE_VOTRE_REPO]
cd a11y-formation

# Configurer les secrets de production (CRUCIAL)
cp .env.example .env
nano .env # Éditez vos mots de passe DB, JWT_SECRET, etc.

# Lancer la stack en production
./deploy-prod.sh
Le script deploy-prod.sh va construire les images optimisées pour le Backend (Next.js) et le Frontend (Nginx).
ÉTAPE 5

Sécurisation SSL (Let's Encrypt)

Pour passer en HTTPS, utilisez Certbot sur l'hôte pour gérer les certificats.

sudo dnf install -y certbot
# Si vous utilisez un reverse proxy externe (ex: Nginx sur le host) :
sudo certbot certonly --standalone -d votre-domaine.fr
CONSEILS PRO
  • Logs : Surveillez avec docker compose -f docker-compose.prod.yml logs -f
  • Backup : Le volume mysql_data est stocké dans /var/lib/docker/volumes/. Configurez un cron pour sauvegarder ce dossier.
  • Monitoring : Utilisez htop et docker stats régulièrement sur le VPS IONOS.
